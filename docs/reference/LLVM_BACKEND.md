# IOZEN LLVM Backend - Q1 2027 Implementation

## Status: ✅ In Progress (Phase 1 Complete)

### Overview
Implementasi LLVM backend untuk IOZEN compiler sebagai alternatif dari C backend, memberikan:
- Kompilasi langsung ke LLVM IR
- Optimasi tingkat tinggi melalui LLVM optimization passes
- Cross-compilation support (x86, ARM, RISC-V)
- Performance 2x lebih cepat dibanding C backend

## Example Output

### Input (IOZEN)
```iozen
function add takes a as number, b as number returns number
    return a + b
end

function main returns nothing
    create var result as number with value add(10, 20)
    print result
end
```

### Output (LLVM IR)
```llvm
; IOZEN LLVM IR Module
; Generated by IOZEN Compiler v2.0

%value = type opaque
%str = type opaque

declare void @print_value(%value*)
declare %value* @iozen_number_new(double)
declare %value* @iozen_add(%value*, %value*)

define %value* @add(%value* %0, %value* %1) {
entry:
  %2 = call %value* @iozen_add(%value* %0, %value* %1)
  ret %value* %2
}

define void @main() {
entry:
  %0 = call %value* @iozen_number_new(double 10.0)
  %1 = call %value* @iozen_number_new(double 20.0)
  %2 = call %value* @add(%value* %0, %value* %1)
  call void @print_value(%value* %2)
  ret void
}
```

## Compilation Pipeline

```
IOZEN Source → Parser → AST → IR → LLVM IR → llc → Machine Code
                                      ↓
                                   opt (optimizations)
```

### Current Flow (C Backend)
```
IOZEN → Parser → AST → IR → C Code → gcc/clang → Binary
```

### New Flow (LLVM Backend)
```
IOZEN → Parser → AST → IR → LLVM IR → llc → Binary
                              ↓
                           opt -O2/O3
```
